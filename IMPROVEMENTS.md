# IP Quality Checker 改进日志

## 改进内容

### 1️⃣ **修复 Windows 窗口闪现问题**

**问题**：PyInstaller 打包的 GUI 应用在启动时如果发生任何异常，窗口会立即关闭，用户看到的是"弹出窗口很快又消失"。

**解决方案**：
- 在 `if __name__ == "__main__":` 块中添加全局异常捕获
- 异常发生时显示一个 GUI 错误对话框（即使是 Windows 也能看到）
- 同时将错误写入日志文件
- 提示用户查看日志文件位置

**新建文件**：
- [logger.py](logger.py) - 日志管理模块

**修改文件**：
- [main.py](main.py) - 添加异常处理和日志集成

---

### 2️⃣ **添加文件日志持久化**

**问题**：日志只存在于 UI 的 Text widget 中，程序崩溃或关闭后日志丢失。

**解决方案**：
- 创建 `logger.py` 模块，提供线程安全的文件日志记录
- 日志自动保存到 `~/.ip-quality-checker/logs/YYYYMMDD.log`
- 每天自动轮转日志文件
- UI 中的日志仍然实时显示（通过回调函数）
- 日志同时写入文件和 UI

**日志位置**：
```
~/.ip-quality-checker/logs/
  ├── 20260511.log (today's log)
  ├── 20260510.log
  └── ...
```

---

### 3️⃣ **确保低分事件被正确记录**

**现有功能**：
- 当评分 < 40 分时，日志记录 `🔴 警报: 评分 XX 低于阈值 40 — 浮窗持续闪烁`
- 如果启用了自动杀死 Claude 进程，会记录清理结果
- 这些日志现在会被持久化到文件

**改进**：
- 低分事件将被永久记录到日志文件
- 用户可以随时查看历史低分事件
- 支持问题诊断和趋势分析

---

### 4️⃣ **设置界面新增日志管理功能**

**新增功能**：
- **日志管理** 部分显示：
  - 日志保存位置
  - 当日日志文件名
  - 说明：所有低于 40 分的事件都会被记录
- **打开日志文件夹** 按钮 - 在文件管理器中打开日志目录
- **打开今日日志** 按钮 - 用默认编辑器打开今天的日志文件
  - macOS: `open -t` (TextEdit)
  - Windows: `notepad.exe`
  - Linux: `xdg-open`

## 使用日志查看低分事件

### 方法 1：通过 UI
1. 点击 **设置** 按钮
2. 滚动到 **日志管理** 部分
3. 点击 **打开今日日志** 按钮
4. 搜索 `警报` 或 `低分` 关键词

### 方法 2：直接打开文件
```bash
cat ~/.ip-quality-checker/logs/$(date +%Y%m%d).log
# 或
grep "警报\|低分\|低分自动清理" ~/.ip-quality-checker/logs/$(date +%Y%m%d).log
```

### 方法 3：通过文件管理器
1. 点击设置中的 **打开日志文件夹**
2. 查看 `.log` 文件

## 异常处理改进

### 场景：Windows 启动失败

**之前**：
- 窗口闪现又消失
- 用户无法看到错误信息

**现在**：
1. 错误被捕获并写入日志
2. 显示一个 GUI 错误对话框，包含：

   - 日志文件位置提示

## 技术细节

### Logger 类特性
- **线程安全**：使用 `threading.Lock` 保护文件写入
- **自动轮转**：检测日期变化，自动创建新日志文件
- **双输出**：同时写入文件和 UI 回调函数
- **错误容错**：文件 I/O 错误不会导致程序崩溃

### 日志路径（平台无关）
```python
LOG_DIR = Path.home() / ".ip-quality-checker" / "logs"
LOG_FILE = LOG_DIR / f"{datetime.now():%Y%m%d}.log"
```

### 回调机制
```python
# UI 设置回调
def log_callback(line: str):
    self.log.insert("end", f"{line}\n")
    self.log.see("end")

logger.set_logger_callback(log_callback)

# 程序中记录日志
logger.log("message")  # 同时写入文件和 UI
```
